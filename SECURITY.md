# Security Policy

## Reporting A Vulnerability

Please do not open a public GitHub issue for suspected security vulnerabilities.

Report security issues through GitHub private vulnerability reporting for this repository.

Do not disclose the issue publicly in an issue, discussion, or pull request.

Please include:

- a clear description of the issue
- steps to reproduce
- affected platform(s) or environments
- potential impact
- any suggested mitigation, if known

## Response Expectations

The project will try to acknowledge valid security reports promptly and coordinate a fix before public disclosure when possible.

## Supported Versions

Until a stable release process is established, security support applies only to the latest development version on the default branch.
