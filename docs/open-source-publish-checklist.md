# Open Source Publish Checklist

- Review `git status` before publishing.
- Review tracked files for local artifacts and machine-specific junk.
- Confirm `SECURITY.md` is present and matches the intended reporting process.
- Confirm `.github/dependabot.yml`, issue templates, and pull request templates are ready for public use.
- Enable Dependabot alerts on GitHub.
- Enable secret scanning if available.
- Decide whether to enable private vulnerability reporting.
- Review `README.md`, repository description, and license details before publishing.
